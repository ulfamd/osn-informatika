# Soal 17 — Menyimpan Kotak Penyimpanan

## Soal

Terdapat 10 kotak kardus dengan panjang sisi:

```txt
1, 1, 2, 2, 2, 3, 3, 4, 6, 8
```

Sebuah kotak:
- dapat dimasukkan ke dalam kotak lain,
- jika panjang sisi kotak di dalam
  lebih kecil dari kotak luar.

Pak Dengklek ingin:
- setiap kotak jika dibuka
  hanya terlihat maksimal satu kotak di dalamnya,
- dan jumlah kotak yang terlihat dari luar
  seminimum mungkin.

Pertanyaannya:

> Berapa banyak kotak kardus paling sedikit
> yang mungkin terlihat dari luar?

Tuliskan jawaban dalam bentuk ANGKA.

:contentReference[oaicite:0]{index=0}

---

## Cara Berpikir

Karena:
```txt
kotak dengan ukuran sama
tidak bisa saling masuk
```

maka kita perlu menyusun:
```txt
rantai kotak
```

dengan ukuran:
```txt
strictly increasing
```

---

# Data Kotak

```txt
1, 1,
2, 2, 2,
3, 3,
4,
6,
8
```

---

# Strategi Penyusunan

Kita susun sebanyak mungkin
ke dalam rantai-rantai.

---

## Rantai 1

```txt
1 → 2 → 3 → 4 → 6 → 8
```

---

## Rantai 2

```txt
1 → 2 → 3
```

---

## Rantai 3

```txt
2
```

---

# Hasil

Terdapat:
```txt
3 rantai
```

Artinya:
- ada 3 kotak terluar
  yang terlihat dari luar.

---

# Mengapa Tidak Bisa 2?

Karena:
```txt
ukuran 2 muncul 3 kali
```

Sedangkan:
- satu rantai hanya boleh memuat
  maksimal satu kotak ukuran 2.

Maka:
```txt
minimal harus ada 3 rantai.
```

---

# Jawaban

3

---

## Catatan Penting OSN

Soal ini melatih:
- greedy
- chain decomposition
- sorting

Konsep penting:

Jika:
```txt
ukuran harus strictly increasing
```

maka:
```txt
jumlah minimum rantai
=
frekuensi maksimum
ukuran yang sama.
```
